I Have a Coolify Instance

aaronwatts@dev 20 April 2026

I'm like a dog chasing cars at this point. I'm running out of fingers to have in pie's. I started learning PHP a while back, after deciding it might actually be a better skill to have over NodeJS and Python in terms of building a back end. That kind of faded off a bit with other projects popping up here and there. But I had seen Coolify and Dokploy, and had intended on setting one up on a VPS to get myself back onto learning PHP.

Of course, you can run PHP directly on a LAMP stack, which I'm interested in learning a bit more about too, but having a control centre where I can easily manage lots of different types of applications certainly had high appeal to me.

I almost moved my NextCloud instance over to Hetzner from Hostinger, with the intention of keeping the Hostinger VPS for either a Coolify or Dokploy instance - Hostinger throttles with high traffic while Hetzner charges extra money for it (apparently, maxed out traffic on a Hetzner VPS wouldn't put you too far out of pocket, but I don't really know enough about backend development to feel confident in protecting against things like that). That fell through, as my Hostinger email wouldn't play nice with the Hetzner NextCloud instance, and I couldn't be bothered to work out why so I just kept my cloud on Hostinger.

Having my own Platform As A Service instance still occupied a space in my mind though. I didn't want to pay for a second Hostinger VPS for something that, at least for now, is just experimental. But recently, my setup at home has been moving around. A Raspberry Pi 500+ has replaced my Raspberry Pi 5 as my desktop. That Raspberry Pi 5 has replaced a Raspberry Pi 4 as my smart TV. And now I had a Raspberry Pi 4, with a 256GB M.2. SATA SSD in a tidy Argon One case with nothing to do. It was time.

I researched running a Raspberry Pi based server and how to expose it via a Cloudflare Tunnel, and got to work setting up my own PaaS server at home. This approach only ended up costing me about £10 for a new extension lead and an ethernet cable as I had everything else that I needed already. I'm not sure what it will do to my electricity bills, it's not particularly power hungry, but it's always on so we'll see.

I went with Coolify instead of Dokploy for a few reasons. The main reason being that the Coolify website had installation instructions for Raspberry Pi, so I don't need to concern myself with hardware compatibility. Coolify will run on any Raspberry Pi that can run a 64 bit operating system - so model 3B and onwards. Another reason was that the Dokploy website is heavy. Both Coolify and Dokploy's websites make my Pi 5 chug, and stress the GPU on rendering. Dokploy's is considerably worse, and I was concerned about how the UI would run on my computers browser. Finally, Coolify just seemed that little bit more beginner friendly to me.

Setting it up wasn't difficult. But I did find it to be a little overwhelming at times. Just following the docs get's you through easily enough. But even compared to setting up NextCloud up on an Ubuntu server where it was a completely new experience to me, I still kind of recognised where I was and what I was doing in the terminal for a typical LAMP install, with Coolify however, it felt like everything was new and completely unknown to me. Needless to say I got through it.

So far I've just set up my tunnels, deployed a static website to test deployment, and set up a WordPress website. Once everything was set up correctly, the WordPress website was as simple as just pressing a button. I've had editing priveleges on a WordPress website before, but this was my first time setting one up from scratch, and from a Coolify standpoint, I have to say I am impressed with how simple it was. I'm yet to experience plugin hell .. hopefully soon.
